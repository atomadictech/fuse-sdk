# Use Cases

Fuse SDK is useful in both developer-facing and agent-facing workflows.

## 1. Repository triage

Use `scan`, `catalog`, `search`, and `status` to understand a codebase before synthesis or emit steps.

## 2. Intent-driven search

Use `search_intent`, `compose_stack`, `logic_map`, `show`, and `explain_block` when you need to find relevant building blocks from a natural-language goal.

## 3. Verification-first automation

Use `verify_block`, `validate`, `doctor`, `status`, and `usage_stats` when downstream systems need explicit health and traceability signals.

## 4. Store maintenance

Use `lint_store`, `validate_store`, `deduplicate_store`, `rebuild_indexes`, `promote_hypotheses`, and `promote_candidate` in maintenance workflows or CI.

## 5. MCP agent integration

Use `fuse-mcp` when you want IDEs or agent runtimes to call Fuse without building a custom adapter.

## 6. Quickstart and bootstrap flows

Use `init`, `seed-info`, and the bundled seed logic-base when you want a fast local starting point for experiments or demos.
